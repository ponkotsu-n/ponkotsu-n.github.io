Dockerで楽々 Database

2017/10/11(Wed)

はじめに

ローカル環境で簡単にデータベースを立ち上げるための、Dockerファイルを作成します。 データベースの起動には、Dockerを使用します。

データベースの作成

データベースの作成方法を、PostgresqlとMySQLの場合について記述します。

Postgresql

データベースは、/var/lib/postgresql/dataに作成します。以下の例では、データベースは以下の内容で作成します。必要であれば、適宜変更してください。

  • データベース名: workdb
  • ユーザ名: postgres
  • パスワード: postgres

docker-compose.ymlは、 となります。

MySQL

データベースは、/var/lib/mysqlに作成します。以下の例では、データベースは以下の内容で作成します。

  • データベース名: workdb
  • ユーザ名: mysql
  • パスワード: mysql

docker-compose.ymlは、 となります。

データベースの起動

データベースの起動するには、ローカル環境に Dockerのイメージファイルを作成します。

Docker Quickstart Terminalを起動し、docker-compose.ymlファイルのあるディレクトリに移動した後、以下のコマンドを実行します。

$ docker-compose build

イメージファイルが作成できたら、データベースを起動します。

$ docker-compose up -d

データ・ボリューム

作成したデータベースは、Dockerコンテナのデータ・ボリューム上で管理します。データ・ボリュームは、Dockeホストの /var/lib/docker/volumes/に作成されます。

まとめ

このように Dockerを使用するとローカル環境にデータベースを手軽に作成できるので、大変便利です。あとは、お好きな SQL Clientからアクセスしてみてください。
